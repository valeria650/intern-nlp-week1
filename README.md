 Submission Workflow

1. Fork this repository to your GitHub account.
2. Clone your fork to your computer.
3. Create a new branch for every assignment or feature.

Example:

```
week01
week02
cnn-project
final-project
```

4. Make your changes **only inside your assigned folder**.
5. Commit your work with meaningful commit messages.
6. Push your branch to your fork.
7. Open a Pull Request to this repository.

---

**Git Workflow**

Create a branch

```bash
git checkout -b week01
```

Stage changes

```bash
git add .
```

Commit

```bash
git commit -m "Complete Week 1 assignment"
```

Push

```bash
git push origin week01
```

Submit a Pull Request through GitHub.

---

**Commit Message Guidelines**

Good examples

```
Add Week 1 assignment

Implement CNN image classifier

Fix preprocessing script

Update project documentation
```

Avoid

```
update

changes

work

final
```

Your commit message should clearly describe what you changed.

---

**Coding Standards**

Please ensure that your work:

- Is well organized.
- Includes comments where necessary.
- Uses meaningful variable and function names.
- Removes unnecessary or unused code.
- Is properly formatted.
- Includes documentation where appropriate.

---

**Jupyter Notebook Guidelines**

When submitting notebooks:

- Clear unnecessary outputs before committing.
- Add markdown cells explaining your work.
- Include a short conclusion.
- Keep notebooks organized and readable.

---

**Folder Organization**

Keep your folder clean.

Example

```
Week01/

dataset/

images/

notebook.ipynb

README.md
```

Avoid placing unrelated files together.

---




** Do NOT**

- Modify another student's work.
- Delete repository files.
- Upload passwords or API keys.
- Upload extremely large datasets unless instructed.
- Commit virtual environments.
- Commit unnecessary generated files.

---



---

**File Naming**

Use clear names.

Good

```
week01_assignment.ipynb

cnn_model.py

report.pdf

README.md
```

Avoid

```
new.ipynb

work.py

finalfinal.ipynb

test2.py
```
**Before Opening a Pull Request**

Confirm that:

- [ ] My work is inside my assigned folder.
- [ ] I created a new branch.
- [ ] My code runs successfully.
- [ ] My notebook is clean and documented.
- [ ] My commit messages are meaningful.
- [ ] I did not modify another student's files.
- [ ] I have reviewed my work.
